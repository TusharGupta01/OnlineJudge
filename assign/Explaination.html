First of all let's talk about the data base used for this platform.



Let's see point wise.

1. User table is for storing the data, personal to particular user.
2. Admin table is for storing the data, personal to particular Admin and it is isolated from user interface.
3. Problem table will be storing the problem id and problem name.
4. Problem Tag has the tags for each problem, it will contain content like dynamic programming, graph problem etc.
5. Problem attempted is having all the detail about that code.
6. Solution table contains the problems solved solved by a particular user.


Now that we have an idea about the database let's have a look of on our website.

Dashboard of a user

This is user dashboard and this page will be shown to user when he/she will login to the account. As you can see there are two buttons, first one is to view the problem currently present on the platform and the second one is to view all the problems that the user has submitted with the result.



Dashboard of admin

This is admin dashboad and will be shown to admin when he/she will login to the account. From this page admin can go to the page where he can add the problem.



Problem adding page

As you can see in the image on the left, this page is to add problem to the database. Admin can insert various features like max amount of space and time for that particular problem and can upload problem with its test cases.



User submission page

This is the page where user can see the problems and after clicking on any one of them user can see the problem and further can submit it to get the detailed report about his/her code.
